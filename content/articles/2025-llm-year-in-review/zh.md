*2025 年 12 月 19 日*

![unnamed](https://bear-images.sfo2.cdn.digitaloceanspaces.com/karpathy/unnamed.webp)

2025 年是 LLM 取得丰硕进展的一年。以下是我个人关注的、略显意外的“范式转变”列表——这些变化改变了格局，并在概念上脱颖而出。

### 1. 基于可验证奖励的强化学习 (RLVR)

2025 年初，所有实验室的 LLM 生产栈大致如下：

1. 预训练（约 2020 年的 GPT-2/3）
2. 监督微调（约 2022 年的 InstructGPT）
3. 基于人类反馈的强化学习（RLHF，约 2022 年）

这是训练生产级 LLM 的稳定且经过验证的配方。2025 年，基于可验证奖励的强化学习（RLVR）成为事实上新增的主要阶段。通过在多种环境中针对自动可验证的奖励对 LLM 进行训练（如数学/代码谜题），LLM 自发形成了在人类看来像是“推理”的策略——它们学会将问题分解为中间计算，并学会各种问题解决策略，往返推敲以找出答案（详见 DeepSeek R1 论文）。这些策略在之前的范式中很难实现，因为对于 LLM 来说，最佳的推理路径和恢复方式并不明确——它必须通过针对奖励的优化来找到适合自己的方法。

与 SFT 和 RLHF 阶段不同，这两个阶段在计算上都是相对较薄/较短的阶段（轻微微调），RLVR 涉及针对客观（不可作弊的）奖励函数进行训练，这允许更长的优化周期。运行 RLVR 被证明能提供很高的性价比$，这吞掉了原本用于预训练的算力。因此，2025 年的大部分能力进步都是由 LLM 实验室消化这个新阶段的余量推动的，总体上我们看到了规模相近的 LLM，但 RL 运行时间更长。此外，这个新阶段还带来了一个全新的旋钮（及其相关的缩放定律）——通过生成更长的推理链和增加“思考时间”来控制测试时计算的能力。OpenAI o1（2024 年底）是 RLVR 模型的首次展示，但 o3 发布（2025 年初）是明显的拐点，你可以直观地感受到其中的差异。

### 2. 幽灵与动物 / 锯齿状智能

2025 年，我（以及我认为业界其他人）首次开始在更直观的意义上内化 LLM 智能的“形态”。我们不是“进化/成长的动物”，而是“召唤幽灵”。LLM 栈的方方面面都不同（神经架构、训练数据、训练算法，尤其是优化压力），所以我们正在获得的智能空间中的实体是非常不同的，用动物的视角来思考它们是不合适的。从监督比特的角度来看，人类神经网络是为丛林中部落的生存而优化的，但 LLM 神经网络是为模仿人类的文本、在数学谜题中收集奖励以及在 LM Arena 上获得人类点赞而优化的。由于可验证领域允许 RLVR，LLM 在这些领域附近“飙升”能力，总体上呈现出有趣的性能特征——它们同时是天才通才和困惑且认知障碍的小学生，随时可能被越狱攻击欺骗而泄露你的数据。

![G6zymj4a0AMNJkJ](https://bear-images.sfo2.cdn.digitaloceanspaces.com/karpathy/g6zymj4a0amnjkj.webp)（蓝色是人类智能，红色是 AI 智能。我喜欢这个版本的迷因（抱歉我丢失了它在 X 上原始帖子的引用），因为它指出人类智能也有自己不同的锯齿状特征。）

与此相关的是，2025 年我对基准测试普遍冷漠并失去了信任。核心问题在于基准测试几乎按其构造就是可验证的环境，因此立即容易受到 RLVR 及通过合成数据生成的较弱形式的影响。在典型的“刷榜”过程中，LLM 实验室团队不可避免地会构建与基准测试占据的嵌入空间小区域相邻的环境，并发展出覆盖它们的锯齿。在测试集上训练成了一门新的艺术。

### 3. Cursor / LLM 应用的新层

我觉得 Cursor 最值得注意的（除了它今年的飞速崛起）是它令人信服地揭示了“LLM 应用”的新层——人们开始谈论“X 的 Cursor”。正如我在今年的 Y Combinator 演讲中强调的（[文字稿](https://www.donnamagi.com/articles/karpathy-yc-talk) 和 [视频](https://www.youtube.com/watch?v=LCEmiRjPEtQ)），像 Cursor 这样的 LLM 应用为特定垂直领域捆绑和组织 LLM 调用：

1. 它们做“上下文工程”
2. 它们在幕后编排多个 LLM 调用，串成越来越复杂的 DAG，仔细平衡性能和成本权衡
3. 它们为循环中的人类提供特定于应用的 GUI
4. 它们提供“自治滑块”

2025 年有很多讨论关于这个新的应用层有多“厚”。LLM 实验室会捕获所有应用，还是 LLM 应用有绿色牧场？个人猜测 LLM 趋势是培养一般能力的大学生，但 LLM 应用将通过提供私有数据、传感器和执行器以及反馈循环来组织、微调和真正激活专业团队进入特定垂直领域的部署专业人士。

### 4. Claude Code / 生活在你的电脑上的 AI

Claude Code (CC) 首次展示了一个 LLM Agent 的样子——一种以循环方式串联工具使用和推理进行扩展问题解决的方法。此外，CC 值得注意的是它在你的电脑上运行，拥有你的私有环境、数据和上下文。我认为 OpenAI 错了，因为他们将早期的 codex/agent 工作重点放在由 ChatGPT 编排的云端容器部署上，而不是简单地本地运行。虽然在云端运行的 agent  swarm 感觉像是“AGI 终局”，但我们生活在一个能力提升足够缓慢和中间的世界里，直接在开发者电脑上运行 agent 更有意义。需要注意的主要区别不在于“AI ops”在哪里运行（云端、本地或其他），而在于其他一切——已经存在并启动的电脑、它的安装、上下文、数据、秘密、配置和低延迟交互。Anthropic 正确地获得了这个优先顺序，并将 CC 打包成一种可爱的极简 CLI 形式，改变了 AI 的样子——它不仅仅是你像 Google 一样去的网站，而是一个生活在你的电脑上的小精灵/幽灵。这是一种与 AI 交互的新范式。

### 5. 氛围编程

2025 年是 AI 跨越必要能力阈值的一年，可以通过简单的英语构建各种令人印象深刻的程序，忘记代码的存在。有趣的是，我在 [这条淋浴时的推文](https://x.com/karpathy/status/1886192184808149383) 中完全无意识地创造了“氛围编程”这个词，没想到它会走多远 :)。通过氛围编程，编程不再是严格保留给训练有素的专业人士，任何人都可以做到。在这层面上，它是我在 [赋能普通人：LLM 如何改变技术扩散的规则](https://karpathy.bearblog.dev/power-to-the-people/) 中写到的另一个例子，说明（与迄今为止所有其他技术截然不同）普通人从 LLM 中获益比专业人士、公司和政府多得多。但氛围编程不仅赋能普通人接近编程，它还赋能训练有素的专业人士编写更多（氛围编程的）软件，否则这些软件永远不会被编写。在 nanochat 中，我用氛围编程在 Rust 中编写了自己的自定义高效 BPE 分词器，而不是必须采用现有库或在该级别学习 Rust。我今年氛围编程了许多项目，作为我想要存在的快速应用演示（例如参见 [menugen](https://karpathy.bearblog.dev/vibe-coding-menugen)、[llm-council](https://github.com/karpathy/llm-council)、[reader3](https://github.com/karpathy/reader3)、[HN 时间胶囊](https://github.com/karpathy/hn-time-capsule)）。而且我氛围编程了整个临时应用只是为了找一个 bug，为什么不呢——代码突然变得免费、临时、可塑、用后即弃。氛围编程将重塑软件并改变职位描述。

### 6. Nano banana / LLM GUI

Google Gemini Nano banana 是 2025 年最令人难以置信的范式转变模型之一。在我的世界观中，LLM 是下一个主要的计算范式，类似于 1970 年代、80 年代等的计算机。因此，我们将看到出于根本相似原因的类似创新。我们将看到个人计算、微控制器（认知核心）或互联网（代理网络）等的等价物等。特别是在 UI/UX 方面，“聊天”与 LLM 有点像 1980 年代向计算机终端发出命令。文本是计算机（和 LLM）的原始/首选数据表示，但它不是人们喜欢的格式，尤其是在输入时。人们实际上不喜欢阅读文本——它既慢又费劲。相反，人们喜欢以视觉和空间方式消费信息，这就是传统计算中发明 GUI 的原因。同样，LLM 应该以我们喜欢的格式与我们交谈——图像、信息图、幻灯片、白板、动画/视频、网络应用等。当然，早期和现在的版本是表情符号和 Markdown 等，它们是“打扮”和布局文本以便于消费的方式，包含标题、粗体、斜体、列表、表格等。但到底谁来构建 LLM GUI？在这个世界观中，nano banana 是这可能是什么样子的第一个早期提示。而且重要的是，它的一个显著方面不仅仅是图像生成本身，而是来自文本生成、图像生成和世界知识的联合能力，全部纠缠在模型权重中。

* * *

**摘要**。2025 年是 LLM 激动人心且略显意外的一年。LLM 正在成为一种新型智能，同时比我预期的聪明得多，也比我预期的笨得多。无论如何，它们非常有用，我不认为业界即使在目前的能力下也实现了其潜力的 10%。同时，有如此多的想法可以尝试，从概念上讲，这个领域感觉广阔开放。正如我今年早先在 [Dwarkesh 播客](https://www.dwarkesh.com/p/andrej-karpathy) 上提到的，我同时（表面上矛盾地）相信我们将看到快速和持续的进步，同时也还有很多工作要做。系好安全带。
