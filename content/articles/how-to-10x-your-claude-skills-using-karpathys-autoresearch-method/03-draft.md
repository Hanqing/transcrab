## 这方法的来源

Andrej Karpathy（OpenAI 联合创始人、特斯拉前 AI 主管、提出"氛围编程"的人）发布了一个叫 autoresearch 的方法。

这个想法很简单：不是你手动改进某样东西，而是让一个 AI Agent 为你循环做这件事。

它尝试一个小改变。检查结果是否变好了。如果变好了就保留，如果没变好就扔掉。

然后它再试一次。再试一次。

他把这个方法用在机器学习代码上。但这个方法适用于任何你可以测量和改进的东西。

包括你在 Claude 中构建的 Skills。

我把这个方法变成了一个在 Claude Code 和 Cowork 中都能用的 Skill。我可以在我的任何其他 Skill 上运行它。

我说"在着陆页 Skill 上运行 autoresearch"，它就会处理整个过程。

## 一个循环如何自动改进你的 Skills

可以这样想。

你有一个配方，10 次里有 7 次做出来很棒。其他 3 次有些不对劲。可能是酱料太淡，可能是调味不对。

不是从头重写整个配方，你改变一种原料。用那个改变烹饪 10 次。

- 变好了？保留这个改变。
- 变坏了？把旧原料放回去。
然后改变下一件事。再烹饪 10 次。变好还是变坏？保留还是恢复。

经过 50 轮这个过程，你的配方 10 次里有 9.5 次成功。

这正是 autoresearch 对你的 Skills 做的事情。

- "配方"就是你的 Skill 提示词。
- "烹饪"就是运行这个 Skill。
- "品尝"就是给输出打分。
你唯一需要提供的就是评分标准。

## 告诉 Agent 什么才是"好"的检查清单

你给 Agent 一个简单的检查清单，定义"好"是什么样子。这是整个过程中你唯一需要做的事情。

你用一个简单的 yes/no 问题列表来做这个。

每个问题检查输出的一个具体方面。通过或失败。就这么多。

Agent 用这个检查清单给每个输出打分，这些分数告诉它它的改变是在帮助还是在伤害。

想象一下像老师用检查清单给论文打分。

但不是"给写作质量打 1-10 分"（这很模糊，每次都不一样），检查清单上的每项都是清晰的 yes 或 no：

- 学生包含论点陈述吗？是或否。
- 每个来源都引用了吗？是或否。
- 不到 5 页吗？是或否。
用这个检查清单你可以给 100 份论文打分，每次都得到一致的结果。

这里也是同样的想法。对于着陆页文案 Skill，你的检查清单可能看起来像：

- "标题包含具体数字或结果吗？"（捕捉像"发展你的业务"这样模糊的标题）
- "文案没有流行词汇如 'revolutionary'、'synergy'、'cutting-edge'、'next-level' 吗？"
- "CTA 使用具体的动词短语吗？"（捕捉像"了解更多"或"点击这里"这样弱的 CTA）
- "第一行提出具体的痛点吗？"（捕捉像"在当今快节奏的世界里。.."这样通用的开头）
- "总文案在 150 字以下吗？"（捕捉那些会流失读者的冗长页面）
你不需要自己弄清楚这些。当你启动 autoresearch 时，Agent 会引导你。

它会问好的样子是什么，帮助你把你的感觉变成具体的 yes/no 问题，甚至会从你已有的风格指南中提取内容。

3-6 个问题是最佳点。超过这个数量，Skill 就会开始玩转检查清单（就像一个不理解材料却记住答案的学生）。

## 运行方法如下

第一步：下载 Skill。在[这里](https://www.dropbox.com/scl/fi/57v11vtj9gzqz10ybv7or/autoresearch.zip?rlkey=f0zbieol7beeykn04erun79ot&dl=1)获取。放到 Claude Code 或 Cowork 的 skills 文件夹里。

第二步：选择一个要改进的 Skill。说"在我的 [Skill 名称] Skill 上运行 autoresearch"。选那个最让你恼火的。那个你一半时间输出很棒另一半时间得到垃圾的。

第三步：Agent 问你 3 件事。要优化哪个 Skill。要用什么测试输入（比如"为一个 AI 生产力工具写着陆页文案"）。还有你的检查清单问题是什么。

第四步：它运行你的 Skill 并显示你的起始分数。这是基线。我的着陆页 Skill 从 56% 开始。模糊的标题、流行词汇汤、弱的 CTA。超过一半的检查都失败了。

第五步：它在你的浏览器中打开一个实时仪表盘。分数随时间上升的图表。每个检查问题的通过/失败分解。每次尝试的更改日志。每 10 秒自动刷新。

第六步：走开。Agent 进入循环。分析什么在失败。对 Skill 提示词做一个小改变。再测试。如果分数上升就保留这个改变，如果下降就撤销。

然后再做一次。再做一次。它会自主持续运行，直到你停止它或连续三次达到 95%+。

你可以看着仪表盘或完全走开。它不需要你运行。而且它把改进后的版本保存为单独的文件，所以你的原始 Skill 保持不变。

## 我的着陆页 Skill 发生了什么

我在我的着陆页文案 Skill 上运行了它。结果如下：

56% → 92%。4 轮更改。3 个保留，1 个撤销。

这是 Agent 实际上在我的 Skill 提示词中改变的：

- 为最常见的失败添加了一个具体规则："你的标题必须包含具体的数字或结果。永远不要使用像'改变你的业务'这样模糊的承诺。"
- 添加了禁用流行词汇列表："永远不要使用：revolutionary、cutting-edge、synergy、next-level、game-changing、leverage、unlock、transform。"
- 添加了一个强有力的着陆页部分的示例，突出了痛点开头和 CTA，这样 Skill 就能看到好的样子而不是猜测。
- 尝试了更严格的字数限制，但因为文案变得太单薄且 CTA 受损而撤销了。（系统会捕捉那些单独看似乎是改进但会伤害整体输出的更改。）
完成后，我得到了：

- 改进后的 Skill，单独保存（原始版本保持不变，以防你想恢复）
- 显示每轮分数的结果日志
- 更改日志，解释每次尝试的更改、Agent 为什么会尝试它、以及它是否有帮助
- 我原始 Skill 的备份，以防我任何时候想回去
那个更改日志可能是最有价值的部分。它是关于那个具体 Skill 什么有效什么无效的完整记录。

当更聪明的模型将来出现时，你把那个更改日志交给它们，它们会从上一个 Agent 停止的地方继续。

## 这在 Skills 之外的东西上也有效

这个方法适用于任何你可以打分的东西。

- 网站速度：有人把它用在页面加载时间上。更改一件事，测量速度，保留或恢复。67 轮从 1100ms 降到 67ms。
- 冷外展：定义你的检查清单："它提到潜在客户的公司吗？它在 75 字以下吗？它以具体问题结尾吗？"让 Agent 运行 50 个变体。
- 时事通讯开头："开头包含个人细节吗？"和"没有陈词滥调短语吗？"让 Agent 自动收紧你的写作。
- 任何你重复使用的提示词
如果你能给它打分，你就能用它做 autoresearch。
