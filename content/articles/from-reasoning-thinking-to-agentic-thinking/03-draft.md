# 从"推理"思维到"Agent"思维

过去两年重塑了我们评估模型的方式以及对模型的期望。OpenAI 的 o1 表明"思考"可以成为一等能力，可以训练并暴露给用户。DeepSeek-R1 证明了推理风格的后训练可以在原创实验室之外复现和扩展。OpenAI 将 o1 描述为一个用强化学习训练的模型，会"在回答之前思考"。DeepSeek 将 R1 定位为与 o1 有竞争力的开放推理模型。

那个阶段很重要。但 2025 年上半年主要是关于推理思维：如何让模型花费更多的推理时计算，如何用更强的奖励训练它们，如何暴露或控制额外的推理努力。现在的问题是接下来会发生什么。我相信答案是 Agent 思维：为了行动而思考，同时与环境交互，并根据来自世界的反馈持续更新计划。

## 1. o1 和 R1 的崛起实际上教会了我们什么

第一波推理模型教会我们，如果我们想在语言模型中扩展强化学习，我们需要确定性、稳定且可扩展的反馈信号。数学、代码、逻辑和其他可验证领域成为核心，因为这些环境中的奖励比通用的偏好监督强得多。它们让 RL 优化正确性而不是合理性。基础设施变得至关重要。

一旦模型被训练来通过更长的轨迹推理，RL 就不再是监督微调的轻量级附加组件。它变成了一个系统问题。你需要大规模的滚动、高吞吐量的验证、稳定的策略更新、高效的采样。推理模型的出现既是建模故事，也是基础设施故事。OpenAI 将 o1 描述为用 RL 训练的推理线，DeepSeek R1 后来通过展示基于推理的 RL 需要多少专门的算法和基础设施工作来强化这一方向。第一个重大转变：从扩展预训练到扩展推理的后训练。

## 2. 真正的问题从来不仅仅是"合并思考与指令"

2025 年初，我们在 Qwen 团队的许多人脑海中有一个雄心勃勃的图景。理想的系统将统一思考模式和指令模式。它将支持可调节的推理努力，精神上类似于低/中/高推理设置。更好的是，它会自动从提示和上下文中推断出适当的推理量，这样模型就可以决定何时立即回答，何时思考更长时间，何时在真正困难的问题上花费更多计算。

从概念上讲，这是正确的方向。Qwen3 是最清晰的公开尝试之一。它引入了"混合思维模式"，在一个系列中同时支持思考和非思考行为，强调可控的思考预算，并描述了一个四阶段的后训练流水线，在长 CoT 冷启动和推理 RL 之后明确包括"思考模式融合"。

但合并比描述起来要难执行得多。困难的部分是数据。当人们谈论合并思考和指令时，他们通常首先想到模型侧的兼容性：一个检查点能否支持两种模式，一个聊天模板能否在它们之间切换，一个服务栈能否暴露正确的切换。更深层次的问题是，两种模式的数据分布和行为目标是实质性不同的。

在尝试平衡模型合并与改进后训练数据的质量和多样性时，我们并没有把一切都做对。在那个修订过程中，我们也密切关注用户实际上如何参与思考模式和指令模式。一个强大的指令模型通常因直接性、简洁性、格式合规性、在重复性高容量企业任务（如重写、标注、模板化支持、结构化提取和运营 QA）上的低延迟而获得奖励。一个强大的思考模型因在困难问题上花费更多 tokens、保持连贯的中间结构、探索替代路径以及保留足够的内部计算来有意义地提高最终正确性而获得奖励。

这两个行为档案相互拉扯。如果合并的数据没有被仔细策展，结果通常是两个方向都很平庸："思考"行为变得嘈杂、臃肿或不够果断，而"指令"行为变得不够干脆、不够可靠，并且比商业用户真正想要的更昂贵。

分离在实践中仍然有吸引力。2025 年晚些时候，在 Qwen3 最初的混合框架之后，2507 线发布了独立的指令和思考更新，包括单独的 30B 和 235B 变体。在商业部署中，大量客户仍然希望为批量操作获得高吞吐量、低成本、高度可控的指令行为。对于这些场景，合并并不是明显的好处。分离生产线允许团队更干净地专注于解决每种模式的数据和训练问题。

其他实验室选择了相反的路线。Anthropic 公开主张集成模型哲学：Claude 3.7 Sonnet 被介绍为混合推理模型，用户可以选择普通响应或扩展思考，API 用户可以设置思考预算。Anthropic 明确表示，他们认为推理应该是一个集成能力，而不是一个单独的模型。GLM-4.5 也公开将自己定位为混合推理模型，同时具有思考和非思考模式，统一推理、编程和 Agent 能力；DeepSeek 后来随着 V3.1 的"思考与非思考"混合推理走向类似的方向。

关键问题是合并是否有机。如果思考和指令仅仅是共存在一个检查点内，但仍然表现得像两个笨拙缝合在一起的人格，产品体验仍然是不自然的。真正成功的合并需要一个平滑的推理努力谱系。模型应该能够表达多个水平的努力，理想情况下在它们之间自适应地选择。GPT 风格的努力控制指向这一点：一个计算上的策略，而不是二元开关。

## 3. 为什么 Anthropic 的方向是一个有用的修正

Anthropic 围绕 Claude 3.7 和 Claude 4 的公开框架是克制的。他们强调集成推理、用户控制的思考预算、现实世界任务、编程质量，以及后来在扩展思考期间使用工具的能力。Claude 3.7 被呈现为具有可控预算的混合推理模型；Claude 4 通过允许推理与工具使用交错来扩展这一点，同时 Anthropic 强调编程、长时间运行的任务和 Agent 工作流作为主要目标。

产生更长的推理轨迹并不会自动让模型更智能。在许多情况下，过度的可见推理信号表明分配能力弱。如果模型试图以同样冗长的方式推理一切，它可能未能确定优先级、未能压缩或未能行动。Anthropic 的轨迹建议了一个更严格的观点：思考应该由目标工作负载塑造。如果目标是编程，那么思考应该帮助代码库导航、规划、分解、错误恢复和工具协调。如果目标是 Agent 工作流，那么思考应该提高长时间跨度内的执行质量，而不是产生令人印象深刻的中散文。

这种对目标实用性的强调指向更大的东西：我们正在从训练模型的时代走向训练 Agent 的时代。我们在 Qwen3 博客中明确表达了这一点，写道"我们正在从专注于训练模型的时代过渡到以训练 Agent 为中心的时代"，并将未来的 RL 进展与环境反馈联系起来用于长跨度推理。Agent 是一个可以制定计划、决定何时行动、使用工具、感知环境反馈、修改策略并在长时间跨度内继续的系统。它由与世界的闭环交互定义。

## 4. "Agent 思维"真正意味着什么

Agent 思维是一个不同的优化目标。推理思维通常由最终答案之前内部审议的质量来判断：模型能否解决定理、写出证明、产生正确的代码或通过基准测试。Agent 思维是关于模型是否可以在与环境交互的同时持续取得进展。

核心问题从"模型能否思考足够长？"转变为"模型能否以一种维持有效行动的方式思考？"Agent 思维必须处理纯推理模型主要可以避免的几件事：

- 决定何时停止思考并采取行动
- 选择调用哪个工具以及以什么顺序
- 从环境中整合嘈杂或部分的观察
- 在失败后修改计划
- 在多轮和多次工具调用之间保持连贯性

Agent 思维是一个通过行动推理的模型。

## 5. 为什么 Agent RL 基础设施更难

一旦目标从解决基准问题转向解决交互任务，RL 栈就会改变。用于经典推理 RL 的基础设施不够。在推理 RL 中，你通常可以将滚动视为主要是相对干净评估器的独立轨迹。在 Agent RL 中，策略嵌入在更大的工具链中：工具服务器、浏览器、终端、搜索引擎、模拟器、执行沙箱、API 层、内存系统和编排框架。环境不再是一个静态验证器；它是训练系统的一部分。

这创造了一个新的系统要求：训练和推理必须更干净地解耦。没有这种解耦，滚动吞吐量就会崩溃。考虑一个必须对实时测试工具链执行生成代码的编程 Agent：推理侧等待执行反馈而停滞，训练侧等待完成的轨迹而饿死，整个流水线的运行效率远低于你从经典推理 RL 预期的 GPU 利用率。添加工具延迟、部分可观察性和有状态环境会放大这些低效。结果是实验在你达到目标能力水平之前很久就变得缓慢和痛苦。

环境本身也成为一等研究工件。在 SFT 时代，我们痴迷于数据多样性。在 Agent 时代，我们应该痴迷于环境质量：稳定性、真实性、覆盖范围、难度、状态多样性、反馈丰富性、抗利用性和滚动生成的可扩展性。环境建设已经开始成为一个真正的创业类别，而不是一个副业项目。如果 Agent 被训练在生产环境中运行，那么环境是核心能力栈的一部分。

## 6. 下一个前沿是更有用的思考

我的期望是 Agent 思维将成为主导的思维形式。我认为它可能最终取代大部分旧式静态独白版本的推理思维：过长的、孤立的内部轨迹，试图通过发出越来越多的文本来补偿交互的缺乏。即使在非常困难的数学或编程任务上，一个真正先进的系统也应该有权利搜索、模拟、执行、检查、验证和修改。目标是稳健地和有成效地解决问题。

训练此类系统最难的挑战是奖励黑客。一旦模型获得有意义的工具访问权限，奖励黑客就变得更加危险。具有搜索能力的模型可能会学会在 RL 期间直接查找答案。编程 Agent 可能会利用仓库中的未来信息、误用日志或发现使任务无效的捷径。具有隐藏泄漏的环境可以使策略看起来超人，而实际上在训练它作弊。这就是 Agent 时代变得比推理时代更加微妙的地方。更好的工具使模型更有用，但它们也扩大了虚假优化的攻击面。我们应该预期下一个严重的研究瓶颈来自环境设计、评估器鲁棒性、反作弊协议以及策略与世界之间更原则性的接口。尽管如此，方向是明确的。工具启用的思考比孤立的思考更有用，并且有更好的机会提高实际生产力。

Agent 思维也意味着工具链工程。核心智能将越来越多地来自多个 Agent 如何组织：一个规划和路由工作的协调器、表现得像领域专家的专门 Agent，以及执行更窄任务的子 Agent，同时帮助控制上下文、避免污染并保持不同推理水平之间的分离。未来是从训练模型到训练 Agent 的转变，以及从训练 Agent 到训练系统的转变。

## 结论

推理浪潮的第一阶段确立了一件重要的事情：当反馈信号可靠且基础设施能够支持时，语言模型之上的 RL 可以产生质量上更强的认知。

更深层的转变是从推理思维到 Agent 思维：从思考更长时间到为了行动而思考。训练的核心对象已经转变。它是模型加环境系统，或更具体地说，Agent 和它周围的工具链。这改变了什么研究工件最重要：模型架构和训练数据，是的，但也包括环境设计、滚动基础设施、评估器鲁棒性以及多个 Agent 协调的接口。它改变了"好的思考"意味着什么：在现实世界约束下维持行动的最有用的轨迹，而不是最长或最可见的轨迹。

它也改变了竞争优势将来自哪里。在推理时代，优势来自更好的 RL 算法、更强的反馈信号和更可扩展的训练流水线。在 Agent 时代，优势将来自更好的环境、更紧密的训练-服务集成、更强的工具链工程以及在一个模型的决策与这些决策产生的后果之间闭环的能力。
