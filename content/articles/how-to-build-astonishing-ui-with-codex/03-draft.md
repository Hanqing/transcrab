# 如何用 Codex 构建惊艳的 UI

GPT-5.4 可以构建漂亮的前端。但只有当你知道如何提问时。

更好的输出始于更紧的约束、视觉参考和真实内容。

以下是如何用 GPT-5.4 构建有意图的前端。

## 为什么 GPT-5.4 默认走向通用

没有具体指导，每个模型都会默认到训练期间看到的最常见模式。

通用的卡片网格。弱的视觉层次。安全、易忘的布局。

GPT-5.4 在前端工作上显著更好。

它生成更具视觉野心的 UI，原生理解图像，可以使用 Playwright 等工具验证和优化自己的输出。

但它仍然需要方向。模型的有意图程度只取决于你的提示。

## 从这里开始：4 个基础

→ 将推理设置为低或中等

反直觉，但这会带来更强的结果。让模型保持快速和专注。

→ 预先定义你的设计系统

排版、调色板、布局规则。在任何代码之前。

→ 提供视觉参考

一张截图、一个情绪板，任何东西。模型从图像中推断节奏、间距和比例。

→ 给它真实内容

真实的文案、真实的产品上下文。占位符文本产生占位符思维。

## 真正有效的设计规则

严格约束模型。OpenAI 自己的入门提示包括这样的规则：

→ 一个构图——第一个视口必须作为一个单一构图阅读

→ 英雄区永远不要卡片。

→ 品牌优先——品牌页面上的产品名称必须是英雄级别

→ 着陆页默认全出血英雄

→ 最多两种字体。一种强调色。

→ 每个区块一个任务。一个目的。一个要点。

石蕊测试：如果你移除英雄图像，页面仍然有效，那图像太弱了。

## 将页面结构化为叙事

在你构建之前，写三件事：

→ 视觉论点——关于氛围和能量的一句话

→ 内容计划——英雄、支持、细节、最终 CTA

→ 交互论点——2-3 个动效想法

然后遵循这个顺序：

1. 英雄——建立身份和承诺

2. 支持图像——展示上下文

3. 产品细节——解释产品

4. 社会证明——建立可信度

5. 最终 CTA——转化

每个区块得到一个任务。如果一个区块试图做两件事，砍掉一个。

## 应用 vs 着陆页

对于应用和仪表盘，适用不同的规则。默认采用 Linear 风格的克制：

→ 平静的表面层次

→ 强大的排版和间距

→ 少颜色

→ 密集但可读的信息

→ 只有当卡片本身就是交互时才用卡片

避免仪表盘卡片马赛克、每个区域的粗边框、装饰性渐变、多个竞争的强调色。

石蕊测试：如果操作员只扫描标题、标签和数字——他们能立即理解页面吗？

## 动效：交付 2-3 个，不是 10 个

动效创造存在感和层次。不是噪音。

挑选三个有意图的动效并坚持使用：

→ 英雄区的一个入场序列

→ 一个滚动链接或粘性效果

→ 一个悬停、揭示或布局过渡

Framer Motion 是推荐的栈。如果一个动效只是装饰性的，移除它。

## 前端技能

OpenAI 发布了一个开源技能，将所有这些编码到一个可重用的提示层中。

用以下命令安装：

$skill-installer frontend-skill

它强制模型在接触任何代码之前定义视觉论点、内容计划和交互论点。值得在下次构建之前运行。

## TL;DR

GPT-5.4 可以生成真正好的前端。但只有当你：

→ 预先约束布局系统

→ 给它视觉参考来工作

→ 将页面结构化为叙事，而不是文档

→ 使用真实内容，不是占位符

→ 保持推理低或中等

模型是有能力的。提示就是设计简报。
